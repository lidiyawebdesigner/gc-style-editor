# GetCourse Style Editor

Chrome-расширение для визуального редактирования стилей блоков GetCourse. Финальный проект курса Claude Code.

**Дедлайн:** ~2026-06-14 (5 дней с 2026-06-09).

## Концепция

- Клик по блоку на странице GetCourse → блок подсвечивается, в боковой панели появляются его данные.
- Контролы в панели (цвета, отступы, тени, радиусы, типографика) + кнопки пресетов из дизайн-системы Лиды.
- Превью применяется на лету.
- Кнопка «Скопировать `<style>`» → готовый CSS для вставки в HTML-виджет GetCourse.

**Без runtime-AI.** Расширение работает офлайн, без API-ключей и подписок. AI (Claude Code) используется только как инструмент разработки.

## Полезные хаки GC (на будущее)

**Cache busting для встроенных виджетов формы.** Если форма GC встроена через скрипт (например в Tilda HTML-блок) и **не показывается повторно** после первой отправки (success-блок не появляется второй раз) — добавь в URL скрипта виджета параметр `&rand=123`. Это ломает кеш браузера, каждый раз грузится свежая форма с чистым DOM.

```
<script src="https://...getcourse.ru/pl/lite/widget/widget?id=ХХХ&rand=123"></script>
```

⚠️ Расширение к этому отношения не имеет — это проблема встраивания виджета, не стилизации. Хорошо иметь в виду при сборке лендингов с формами GC в Tilda.

## Стек

- Vanilla JavaScript, никаких фреймворков
- Chrome Manifest V3
- Content script + injected sidebar (без отдельного popup-окна)

## Структура

```
extension/        — код расширения (грузится в Chrome как unpacked)
  manifest.json   — конфиг
  content.js      — скрипт, инжектящий панель в страницу GetCourse
  sidebar.css     — стили панели
brief/            — HTML типовых блоков GetCourse для тестов
```

## Установка для разработки

1. Открой Chrome → `chrome://extensions/`.
2. Включи **Режим разработчика** (Developer mode) — тумблер справа сверху.
3. Нажми **Загрузить распакованное расширение** (Load unpacked).
4. Выбери папку `extension/` (НЕ корневую папку проекта, а вложенную `extension/`).
5. Открой любую страницу `*.getcourse.ru` — справа сверху появится панель.

После каждого изменения кода → `chrome://extensions/` → кнопка обновления (↻) у расширения → перезагружай страницу GetCourse.

## День 5 v0.9.0 — глобальный селектор кнопки + hover-эффект

Лида прислала большую выгрузку материала с курса GetCourse (`brief/gc-styles-source.md`, 1800 строк). Главное открытие: в курсе кнопки стилизуются глобальным селектором `.btn.f-btn` (без `.lt-form` scope) — то есть один стиль на все кнопки лендинга. Не как у меня в 0.8.x.

### Изменения

**Селектор кнопки в экспорте:** `.lt-form .f-btn` → **`.btn.f-btn`**. Применяется ко всем кнопкам на странице (CTA hero, форма, футер — одинаковый стиль). Применение в DOM при превью остаётся scoped через `FORM_SELECTORS.buttons` внутри формы.

**Hover-эффект кнопки.** Новый dropdown в подгруппе «Кнопка»:
- Нет
- Увеличение → `transform: scale(1.05)`
- Подъём → `transform: translateY(-5px)`
- Сдвиг вправо → `transform: translateX(10px)`

В экспорте два правила:
```css
.btn.f-btn { transition: 0.3s ease-in-out; }
.btn.f-btn:hover { transform: scale(1.05) !important; transition: 0.5s ease-in-out !important; }
```

В DOM превью применяется через `updatePreviewStyle()` — инжектит `<style id="gse-preview-style">` в `<head>`. Pseudo-класс `:hover` нельзя задать inline.

Из материала курса остались за бортом — Лида подтвердила что сейчас не нужно:
- Подгруппа «Карточка формы» (`.lt-normal-form.lt-form-inner.lt-form` — фон/рамка/скругление формы) — Лиде сейчас не надо

- Пресет «Без обводки с подчёркиванием» — не запросила
- Таймер, слайдеры, карточки/плитки, иконки — это другие темы, не формы

При запросе можно добавить отдельной итерацией.

## День 5 v0.8.1 — fix отступов между блоками формы + высота кнопки

**Отступы.** В 0.8.0 я применял `margin-bottom` через `closest(FORM_SELECTORS.fieldGroups)` от инпута. На некоторых формах GC поля не упакованы в `.field-group/.form-group/.builder-item.part-field` — закрывающий контейнер другой. Переписал:
- В DOM: `currentForm.querySelectorAll('.builder-item, .field-group, .form-group, .form-field')` — все возможные обёртки разом, с фильтром `isGcNoise`
- В экспорте: расширил селектор до `.lt-form .builder-item, .form-block .builder-item, .modal-form-inner .builder-item, .builder-item.part-field, .builder-item.part-button` — теперь покрываются все распространённые случаи

**Высота кнопки.** Новый слайдер `formBtnHeight` (30–90 px) в подгруппе «Кнопка». Применяет:
- `height`
- `min-height` (чтобы GC `min-height: 60px` из media-queries не перебивал)
- `line-height: height - 4px` для центрирования текста
- `padding-top/bottom: 0` чтобы GC-padding не растягивал

`syncFormControls` читает реальную высоту через `getBoundingClientRect().height`.

## День 5 v0.8.0 — селекторы по эталону GC (домашка курса)

Лида прислала эталонный материал из домашки курса по стилизации форм GetCourse. Я переписал ВСЕ селекторы экспорта по их стандарту — должно работать на проде.

### Эталонные селекторы (теперь используются в экспорте)
| Что | Селектор |
|---|---|
| Поле ввода | `.lt-form .f-input` |
| Placeholder | `.lt-form .f-input::placeholder` |
| Focus полей | `.lt-form .f-input:focus, .lt-form .f-input:focus-visible` |
| Autofill (анти-жёлтый Chrome) | `input:-webkit-autofill` |
| Согласие | `.global-confirm-checkbox-block` (главный + старые fallback) |
| Ссылка в согласии | `.global-confirm-checkbox-block a` |
| Кнопка | `.lt-form .f-btn` |
| Отступ полей | `.builder-item.part-field, .builder-item.part-button` |

`.lt-form` scope даёт правильную специфичность без `#ltBlock...` префикса.

### Новые контролы в секции «Форма»

**Поля ввода** (большое расширение):
- Цвет текста полей (color picker) → `color`
- Цвет фона полей (color picker) → `background-color` (отдельно от прозрачного)
- Цвет подсказки (color picker) → `::placeholder { color }`
- Цвет рамки в фокусе (color picker) → `:focus { outline }`
- Отступы по бокам (`formInputPaddingX`, 0–40 px) → `padding: 0 Xpx`
- ☑ Прозрачный фон полей (как было)
- ☑ Убрать жёлтый автозаполнения Chrome (новое, по умолчанию включено) → инжектит `input:-webkit-autofill { ... }` с фоном, совпадающим с фоном полей

**Согласие** (новая подгруппа):
- Размер текста согласия (был)
- Цвет ссылки (color picker) → `.global-confirm-checkbox-block a { color, text-decoration: underline }`

### Preview style для pseudo-элементов

Pseudo-элементы (`::placeholder`, `:focus`, `:-webkit-autofill`, `a`) нельзя задать через inline `style`. Делаю `<style id="gse-preview-style">` инъекцию в `<head>`, обновляю на каждое изменение через `updatePreviewStyle()`. Очищается при reset/выборе новой формы.

### syncFormControls
Читает с первого `.f-input`:
- `paddingLeft` → отступ X
- `color` → цвет текста полей
- `backgroundColor` → цвет фона + флаг прозрачности

При выборе новой формы `clearFormDirty` сбрасывает все dirty-флаги и preview-стиль.

## День 5 v0.7.3 — селекторы по классам (вместо ID) в individual rules

Гипотеза: на публичной странице GC рендерит элементы с другими ID, чем в админке. Из-за этого правила `#button6774760 { background: red }` промахивались. Тех отдел Zerocoder экспортирует через классы (`.f-btn { background: #FD0200 !important }`) — это работает всегда.

**Фикс в `getIndividualExportRules`:** приоритет селектора теперь:
1. Семантический GC-класс через `pickStableClass()` (`.f-btn`, `.f-header`, `.f-text`, `.f-input`, `.lt-block-wrapper`, ...), завёрнутый в scope блока (`#ltBlock... .f-btn`) — чтобы не растекаться на соседние блоки на странице.
2. ID элемента — fallback (если семантического класса нет).

**Дополнительно:** `FORM_SELECTORS.forms` расширен на `form[id^="ltForm"]` — на случай если у формы только GC-ID без распознаваемых классов.

## День 5 v0.7.1 — голый CSS + individual rules без ID

**Главный фикс:** убрана `<style>...</style>` обёртка из экспорта. CSS-поле блока GC ожидает голый CSS — GC сам оборачивает в `<style>`. Раньше получался двойной тег, ничего не применялось.

**Individual rules без ID** — если у тронутого элемента нет своего ID, теперь используется fallback `#blockScope .className` через `pickStableClass()`. Раньше такие элементы (например, `.lt-block-wrapper` с правкой `border-radius`) молча выкидывались из экспорта.

`pickStableClass()` — сначала PRIORITY-классы GC (`f-header`, `f-text`, `f-btn`, `f-input`, `lt-block-wrapper`, `block-box`, `cover-wrapper`, `lt-tsr-block`, `image-card`, ...), потом первый класс не из шумного списка (`animated`, `lazyloaded`, `js-*`, `col-md-*`, `text-center` и т.д.).

**UI:** подсказка под кнопкой теперь говорит «Вставь в CSS-поле блока: Настройки блока → CSS. Без `<style>` тегов».

## День 5 v0.7.0 — Clean-up: чистый семантический экспорт

**Кардинальный фикс по эталону тех отдела Zerocoder.** Лида показала пример: у них формы стилизуются глобальными классами (`.f-input`, `.f-btn`, `.form-control`) одним общим селектором — без `#ltBlock` scope, без `nth-of-type`, без ID-каши. И стили вставляются прямо ВНУТРЬ блока GC (у блока есть встроенное поле для CSS), не в отдельный HTML-блок.

### Что было плохо в 0.6.x
1. **`applyFontToForm` ходил по `querySelectorAll('*')`** и мазал `font-family` на каждый служебный div редактора GC (`.setting-edit-link`, `.add-redesign-subblock`, `.sort-item-handler`, …, hidden UTM-инпуты, `<script>`, `<style>` теги).
2. Экспорт выгружал inline-стили КАЖДОГО тронутого элемента с селекторами типа `#ltBlock... #элемент:nth-of-type(7)` — ~95 правил, 90% мусор.
3. На публичной странице GC служебных классов нет → селекторы промахивались → стили «не применились».
4. Отступы между полями не работали — `closest()` искал `.field-group`, а у GC обёртка `.builder-item.part-field`.

### Что в 0.7.0

**`isGcNoise(el)`** — фильтр служебных элементов GC-редактора. Применяется везде, где обходим DOM формы:
- Классы: `setting-edit-link`, `add-redesign-subblock`, `sort-item-handler`, `copy-item-handler`, `delete-item-link`, `add-item-link`, `common-setting-link`, `form-result-block`, `external-value`, `fa`, `add-redesign-block`, `remove-redesign-block`
- Теги: `<script>`, `<style>`, `<noscript>`, `<link>`, `<meta>`, `<template>`
- Hidden inputs + UTM-инпуты (`utm_source/medium/campaign/...`, `alias`, `url_param`, `ym_uid`, `transaction_id`)
- Любой потомок служебного элемента (через `closest`)

**`applyFontToForm` переписан** — не идёт по `*`, а только по `FORM_SELECTORS.textTargets` (`input:not([hidden])`, `textarea`, `select`, `button`, `label`, `.f-btn`, `.f-input`, `.f-header/.f-text/...`, `h1-h6`, `p`, `a`). + фильтр `isGcNoise`.

**`applyToFormParts`** — тоже фильтрует через `isGcNoise`. Все формовые слайдеры теперь не цепляют мусор.

**`FORM_SELECTORS.fieldGroups`** — добавлены `.builder-item.part-field, .builder-item.part-button, .builder-item`. Отступ между полями теперь корректно едет на родителя `.builder-item`, а не на сам `<input>`.

### Dirty-tracking
- На каждом слайдере формы: `slider.dataset.dirty = '1'` при изменении значения.
- На dropdown шрифта: `formFontSelect.dataset.dirty = '1'` при `change`.
- Шорткат-кнопки («Поля 100%», «Кнопка 100%», «Авто ширина») тоже метят dirty соответствующий слайдер.
- `clearFormDirty()` — обнуляет всё; вызывается при `syncFormControls` (новая форма) и при reset.

Зачем: экспорт включает ТОЛЬКО то, что Лида правда трогала. Не выгружаем дефолтные значения GC, к которым синхронизировался слайдер при выборе.

### Семантический экспорт

Два источника правил:

1. **`getFormExportRules()`** — по dirty-слайдерам и dirty-селект шрифта. Селекторы — глобальные классы GC (`.f-input`, `.f-btn`, `.builder-item.part-field`, ...). Без `#ltBlock` scope. Делится на семантические группы с комментариями (`/* Поля ввода */`, `/* Кнопка формы */`, ...).

2. **`getIndividualExportRules()`** — для элементов с GC-ID, тронутых через picker (цвет конкретной кнопки, фон конкретной обёртки). Использует `#elementId` селектор. Формовые свойства (`font-family`, `font-size`, `width`, `max-width`, `height`, `margin-bottom`) вырезаются — они уже в формовых правилах.

**`groupRules()`** — слияние правил с одинаковым телом под один селектор-список через запятую.

**Без `#ltBlock` префикса** — стили работают глобально, селекторы стабильны между админкой и публичной страницей. Достаточно вставить `<style>` в встроенное CSS-поле блока GC.

### Пример экспорта (после 0.7.0)
```css
<style>
@import url('https://fonts.googleapis.com/css2?family=Manrope:wght@300;400;500;600;700;800&display=swap');

/* Шрифт формы */
p, a, h1, h2, h3, h4, h5, h6, button, label, input, textarea, select,
.f-btn, .f-input, .form-control, .f-header, .f-subheader,
.f-text, .f-description, .f-name, .html-content {
  font-family: Manrope, sans-serif;
}

/* Поля ввода */
.f-input {
  width: 100% !important;
  max-width: 100% !important;
  height: 40px !important;
  font-size: 13px !important;
}

/* Отступ между полями */
.builder-item.part-field,
.builder-item.part-button {
  margin-bottom: 25px !important;
}

/* Кнопка формы */
.f-btn {
  font-size: 14px !important;
}

#button2548466 {
  color: #fff !important;
  background-color: #4dbee3 !important;
  border-radius: 5px !important;
  font-weight: 600 !important;
  text-transform: uppercase !important;
  letter-spacing: 0.04em !important;
}
</style>
```

5–6 правил вместо 95. Чисто, читаемо, рукой можно поправить.

## День 5 v0.6.1 — экспорт стилей блока + КАПС + fix отступов формы

**Экспорт стилей блока.** Новая секция «Экспорт» в сайдбаре с одной большой кнопкой «📋 Скопировать стили блока». От `selectedElement` поднимается до ближайшего `.lt-block`, обходит все потомки с тронутыми стилями (через WeakMap `originalStyles`), складывает в один `<style>` со scoped-селекторами. Готово для вставки в HTML-виджет GetCourse рядом с блоком.

**Селекторы для экспорта** строит `buildSelector(el, scope)`:
- Если у элемента есть `id` → `#blockId #elementId`
- Иначе — путь от scope через `tagname.className:nth-of-type(N)`, заворачивает в `#blockId ...`
- Фильтр classNames пропускает шумные классы (`animated`, `lazyloaded`, `js-*`, `col-md-*`, утилитарные `text-center` и т.д.)

**Подгруженные Google Fonts** автоматически попадают в экспорт через `@import url(...)` в начале `<style>`.

**КАПС в типографике.** Новый ряд «Регистр» под выравниванием: `Aa` / `КАПС` / `aa`. Применяет `text-transform`; для `uppercase` добавляет `letter-spacing: 0.04em` (по-человечески смотрится). Mirror-to-siblings работает.

**Fix отступов между полями.** Логика: от каждого инпута идём `closest()` к контейнеру (`.field-group / .form-group / .form-field / .field-input / .form-row / .field-text`), применяем `margin-bottom` на него. Если контейнера нет — на сам инпут. Решает кейс, когда в GC форме поля сидят без обёртки.
